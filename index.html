Hello. This is soulheart.

Haven’t you wished there was a super easy way to set up a remote data source for autocomplete?

Soulheart makes it possible to set up a remote data source with zero backend programming. You can deploy to Heroku with the click of a button, upload a tsv with a text column and start using it.

Here is how to deploy the above select, step by step:

  1. Deploy to Heroku by clicking
  2. Install the Heroku CLI and open up a terminal window.
  3. Run this command in the command prompt replace "sh-example-simple" with the name of your app on Heroku: 
    heroku run -a sh-example-simple soulheart load https://raw.githubusercontent.com/sethherr/soulheart/gh-pages/example_sources/manufacturers.tsv
  4. Add a select box to your page, <select id="basic-example-select"></select>, then use this javascript to hook up the select box with select2 Once again, replace "sh-example-simple" with the name of your app on Heroku: 
    $('#sh-example-simple-select').select2({
  allowClear: true,
  placeholder: "Choose a manufacturer",
  ajax: {
    url: 'http://sh-example-simple.herokuapp.com',
    dataType: 'json',
    delay: 250,
    data: function(params) {
      return {
        q: params.term,
        page: params.page,
        per_page: 10
      };
    },
    processResults: function(data, page) {
      return {
        // Select2 requires an id, so we need to map the results and add an ID
        // You could instead include an id in the tsv you add to soulheart ;)
        results: data.matches.map(function(item) {
          return {
            id: item.text,
            text: item.text
          };
        }),

        pagination: {
          // If there are 10 matches, there's at least another page
          more: data.matches.length === 10
        }
      };
    },
    cache: true
  }
});

Sorting by priority in addition to alphanumerically

So, for example, let’s say you want to have bike manufacturers using select2 like the above example.

Some bike manufacturers are more popular than others - and since we expect people to be searching for more popular ones more frequently, it's important to make them show up first — for example, if you input a "t", the first manufacturer to select should be "Trek", not "Tacx".

Items with equal scores are ordered alphanumerically. So in the manufacturers example above, manufacturers are grouped in broad levels of popularity - 10, 100, 250 & 500 - higher numbers show up first.

Set a priority to organize the way items are ordered.

Categories

Search for items in only one category by adding a `categories` parameter:

http://sh-example-simple.herokuapp.com?categories=manufacturer

Search for items in any of the included categories by seperating the categories with commas:

http://sh-example-simple.herokuapp.com/?categories=manufacturer,frame%20manufacturer

Arbitrary return data

Anything in the data hash will be in the return value.

For TSVs and CSVs, any column prefixed with 'data' will be put in the data hash.

Formatting data

You can add data from JSON, CSV and TSV files.

Items can be very simple - all that is required is a text value.

A text column or key is required.

So the simplest thing to load is a csv that looks like this:

Text
Apple
Orange
Pear
Banana

Optional attributes: if you have a priority, category, or a data hash, it will use them as explained above.

Here is an excerpt of the tsv that was loaded in for the first example:

text category priority data-website data-logo
All City Frame manufacturer 100 http://allcitycycles.com https://files.bikeindex.org/uploads/Ma/371/all-city.png
Salsa Frame manufacturer 250 http://salsacycles.com/
Sunday Frame manufacturer 10 http://www.sundaybikes.com
SRAM Manufacturer 250 http://www.sram.com
Surly Frame manufacturer 500 http://surlybikes.com

Results from sh-example-simple.herokuapp.com (the place where the above data is loaded) look like:

{
  "text": "Surly",
  "category":"Frame manufacturer",
  "website":"http://surlybikes.com",
  "logo":null
}

For JSON

Soulheart uses line delineated JSON streams, so it doesn’t have to load the whole file into memory. Which just means - put each object onto a seperate line.

For the simplest case, with just text values in JSON:

{ "text": "Jamis" }
{ "text": "Specialized" }
{ "text": "Trek" }

Here is an example of what a possible hash you could pass is

{ "text": "Jamis", "category": "Bike Manufacturer" }
{ "text": "Specialized" }
{ "text": "Trek" }

If you set text in data, it will respond with that rather than the term it searches by. I haven’t figured out a use case for this yet, but I’m sure one exists.
